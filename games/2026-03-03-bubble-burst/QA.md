# QA - 2026-03-03 バブルバースト

## 対象
- スマホ縦画面（想定幅 360-430px）
- タップ操作

## チェック結果
- 操作性：pointerdown イベントを使用しており、タップの反応は即時。ヒット判定はバブル半径の円形判定で誤タップが起きにくい。最前面のバブル（描画最後 = 配列の末尾）から判定するため複数が重なっても正確に動作する。
- 表示：縦画面全画面表示でスクロールなし。HUDは画面上部に固定（スコア左・タイマー右）。バブルのアニメーション（出現→安定→消滅）がスムーズ。ゲーム終了後のオーバーレイが再表示される。
- パフォーマンス：requestAnimationFrame を使用し canvas 描画を最小化。バブルの最大同時出現数は難易度に依存するが高くても15個程度で軽量。radialGradient とシャドウを毎フレーム生成するのが最大の負荷。

## バグ
- なし（コードレビューの範囲内で検出なし）

## 改善提案（最大3つ）
1. バブル破裂時に効果音（Web Audio API で短いプチっ音）を追加すると爽快感が増す
2. バブルを逃した（自然消滅した）際に画面端で軽く揺れる視覚フィードバックを加えると難易度感が出る
3. スコアが10の倍数になった瞬間にコンボ演出（テキストフラッシュ）を追加するとモチベーションが維持しやすい
